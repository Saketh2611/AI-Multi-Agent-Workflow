### Research & Design

To build a multi-agent workflow with LangGraph and the Gemini API, we will implement a foundational system featuring a `Planner` and a `Coder` agent.

**1. Design the Multi-Agent Workflow**

*   **Agents:**
    *   **Planner Agent:** This agent will receive a user's request and generate a high-level plan, breaking down the request into actionable coding steps. Its role is to understand the problem and outline a solution strategy.
    *   **Coder Agent:** This agent will take the plan generated by the Planner and attempt to write Python code to fulfill the steps. Its role is to translate the plan into executable code.
*   **Shared State Schema:** We'll use a `TypedDict` to define the shared state that passes between agents. This state will include:
    *   `request`: The initial user request provided to the system.
    *   `plan`: The detailed plan generated by the Planner agent.
    *   `code`: The Python code generated by the Coder agent.
*   **Interaction Flow (Transitions):**
    1.  The workflow starts with an initial user `request`.
    2.  The `Planner` agent processes the `request` and updates the shared state with a `plan`.
    3.  The `Coder` agent receives the `plan` from the shared state and generates `code`, updating the state.
    4.  The workflow ends after the `Coder` has successfully produced the code.

**2. Implement LangGraph Structure and Agents**

We will use `langgraph.graph.StateGraph` to define our workflow. Each agent will be represented as a node within this graph.

*   **`AgentState`:** A `TypedDict` will define the structure of the shared state, ensuring type safety and clarity for the data passed between nodes.
*   **Agent Functions:** Each agent (`planner_agent`, `coder_agent`) will be implemented as a Python function. These functions will:
    *   Accept the current `AgentState` as input.
    *   Utilize the `ChatGoogleGenerativeAI` model (e.g., `gemini-pro`) to perform their specific task (planning or coding) based on the relevant parts of the state.
    *   Return a dictionary containing the updates to be applied to the shared `AgentState`.
*   **Graph Definition:**
    *   A `StateGraph` instance will be created, initialized with `AgentState`.
    *   `add_node()` will register the `planner_agent` and `coder_agent` functions as named nodes in the graph.
    *   `set_entry_point("planner")` will designate the `planner` node as the starting point of the workflow.
    *   `add_edge()` will define the sequential flow: `planner` -> `coder`.
    *   `add_edge("coder", END)` will mark the `coder` node as the final step, terminating the workflow.
*   **Compilation:** The `workflow.compile()` method will finalize the graph structure, making it ready for execution.

### Code Implementation

```python
import os
from typing import TypedDict, Annotated
from langchain_google_genai import ChatGoogleGenerativeAI
from langgraph.graph import StateGraph, END
from langgraph.graph.message import add_messages

# Ensure you have your Gemini API key set as an environment variable
# os.environ["GOOGLE_API_KEY"] = "YOUR_GEMINI_API_KEY"

# Install libraries:
# pip install langchain-google-genai langgraph

# 1. Define the Shared State
class AgentState(TypedDict):
    """
    Represents the state of our multi-agent workflow.
    """
    request: str
    plan: str
    code: str

# Initialize the Gemini model
# You can choose other Gemini models like "gemini-1.5-pro-latest"
llm = ChatGoogleGenerativeAI(model="gemini-pro", temperature=0.7)

# 2. Implement Agent Functions (Nodes)

def planner_agent(state: AgentState) -> AgentState:
    """
    The Planner agent takes a request and generates a plan.
    """
    print("---PLANNER AGENT---")
    request = state["request"]
    prompt = f"""You are a helpful AI assistant. Your task is to break down the following user request into a clear, actionable plan for a Python developer.
    Focus on the logical steps required to implement the request.

    User Request: {request}

    Provide the plan as a series of numbered steps.
    """
    response = llm.invoke(prompt)
    plan = response.content
    print(f"Generated Plan:\n{plan}")
    return {"plan": plan}

def coder_agent(state: AgentState) -> AgentState:
    """
    The Coder agent takes a plan and generates Python code.
    """
    print("---CODER AGENT---")
    plan = state["plan"]
    prompt = f"""You are an expert Python programmer. Your task is to write Python code based on the following plan.
    Ensure the code is complete, runnable, and addresses all steps in the plan.
    If the plan involves external libraries, assume they are installed.

    Plan:
    {plan}

    Provide only the Python code, enclosed in triple backticks (```python ... ```).
    """
    response = llm.invoke(prompt)
    code = response.content
    print(f"Generated Code:\n{code}")
    return {"code": code}

# 3. Develop LangGraph Structure

# Create a StateGraph
workflow = StateGraph(AgentState)

# Add nodes for each agent
workflow.add_node("planner", planner_agent)
workflow.add_node("coder", coder_agent)

# Set the entry point
workflow.set_entry_point("planner")

# Define the transitions
workflow.add_edge("planner", "coder")
workflow.add_edge("coder", END) # The workflow ends after the coder generates code

# Compile the graph
app = workflow.compile()

# Example Usage
if __name__ == "__main__":
    # Example 1: Simple calculation
    print("\n--- Running Workflow for Example 1 ---")
    initial_state_1 = {"request": "Write a Python function that calculates the factorial of a given number.", "plan": "", "code": ""}
    result_1 = app.invoke(initial_state_1)
    print("\n--- Final State for Example 1 ---")
    print(f"Request: {result_1['request']}")
    print(f"Plan:\n{result_1['plan']}")
    print(f"Code:\n{result_1['code']}")

    # Example 2: String manipulation
    print("\n--- Running Workflow for Example 2 ---")
    initial_state_2 = {"request": "Create a Python script that reverses a string and counts the number of vowels in it.", "plan": "", "code": ""}
    result_2 = app.invoke(initial_state_2)
    print("\n--- Final State for Example 2 ---")
    print(f"Request: {result_2['request']}")
    print(f"Plan:\n{result_2['plan']}")
    print(f"Code:\n{result_2['code']}")
```

### Explanation of Code

1.  **`AgentState` (Shared State):** This `TypedDict` defines the structure of the information that will be passed between our agents, ensuring a consistent data schema throughout the workflow.
2.  **`ChatGoogleGenerativeAI`:** We initialize the Gemini model (`gemini-pro` in this case) which will power our agents' reasoning and generation capabilities. The `temperature` parameter is set to 0.7 for balanced creativity and coherence.
3.  **`planner_agent` Function:**
    *   Takes the current `state` as input.
    *   Constructs a prompt for the Gemini model, instructing it to act as a planner and break down the user `request` into actionable steps.
    *   Invokes the `llm` with the prompt to get the plan.
    *   Extracts the generated `plan` from the model's response content.
    *   Returns a dictionary `{"plan": plan}` to update only the `plan` field in the shared state.
4.  **`coder_agent` Function:**
    *   Takes the current `state` (which now includes the `plan` generated by the planner) as input.
    *   Constructs a prompt for the Gemini model, instructing it to act as an expert Python programmer and generate code based on the provided `plan`. It emphasizes runnable and complete code.
    *   Invokes the `llm` with the prompt to get the code.
    *   Extracts the generated `code` from the model's response content.
    *   Returns a dictionary `{"code": code}` to update only the `code` field in the shared state.
5.  **`StateGraph` Setup:**
    *   An instance of `StateGraph` is created, explicitly specifying `AgentState` as its state type.
    *   `add_node()` registers our `planner_agent` and `coder_agent` functions as named nodes ("planner" and "coder") in the graph.
    *   `set_entry_point("planner")` indicates that the workflow always begins execution at the `planner` node.
    *   `add_edge("planner", "coder")` defines a direct transition: after the `planner` node completes its task, the workflow automatically moves to the `coder` node.
    *   `add_edge("coder", END)` signifies that after the `coder` node completes, the entire workflow terminates, and the final state is returned.
6.  **`app.invoke()`:** This is the method used to run the compiled LangGraph workflow. You provide an initial `AgentState` dictionary containing the user's `request`, and the graph will execute the agents in their defined sequence, updating the shared state along the way. The final state after all agents have run is returned.
